## [0.0.1.5.ProtoAlpha] — 2026-05-29

### Автоопределение порядка кадров

- Добавлено автоматическое сопоставление фото из папки фотографа с эталонами `reference/Sneakers/search/1.jpg … 8.jpg`.
- Номер кадра (`01` … `08`) назначается по визуальному сходству, а не по имени исходного файла.
- Миниатюры в GUI сортируются по назначенному номеру кадра; на бейджe показывается уверенность matching (например `03 · 90%`).
- Правила автокадрирования применяются по назначенному номеру, даже если исходник назывался иначе.
- Экспорт сохраняет файлы как `1.jpg`, `2.jpg`, … `8.jpg` по назначенному номеру кадра.
- Дроплеты Photoshop маршрутизируются по назначенному `frame`, а не по имени исходника.

### Эталоны

- Основные эталоны для правого блока «Эталон» и визуального контроля: `reference/Sneakers/1.jpg … 8.jpg`.
- Отдельный набор для распознавания порядка: `reference/Sneakers/search/1.jpg … 8.jpg`.
- Автоподстановка эталона в правом углу ищет файл по номеру текущего кадра (`1.jpg` / `01.jpg`), а не по индексу миниатюры.

### Сборка и дистрибутив

- В portable-сборку (`dist\AutoRAWCompressor`) добавлены папка `droplets` и файл `CHANGELOG.md`.
- `build.bat`: пункт **2 — Build MSIX package** и команда `build.bat msix`.
- MSIX собирается через Windows SDK (`build\build_msix.py`): иконки из `favicon.ico`, подпись self-signed сертификатом.
- Установщик включает всё содержимое dist, включая `droplets` и `CHANGELOG.md`.
- Рядом с MSIX создаётся `AutoRAWCompressor-<версия>-CHANGELOG.txt` — краткое описание изменений текущей версии.
- Оптимизация размера: исключены неиспользуемые `numba`/`llvmlite` (~100 MB), лишние PSD/PNG из assets; пакет ~64 MB (цель ≤ 99 MB).

### Автообновление

- Меню «Проверить обновление…»: запрос к GitVerse API, скачивание ZIP с прогрессом, распаковка в папку приложения, перезапуск.
- Токен: `gitverse_token` в `ui_config.json` или переменная `GITVERSE_TOKEN`.
- ZIP для релиза: `python build\build_release_zip.py` → загрузить в Releases на GitVerse.
- `ui_config.json` и `zona/data.dat` при обновлении не перезаписываются.

### Экспорт и обновления (GUI)

- Во время экспорта: кнопки **Пауза** / **Продолжить** и **Отмена** в строке состояния.
- По завершении показывается **рабочее время без пауз**.
- При запуске: окно о новой версии (Установить / пропустить до следующего запуска).

## [0.0.1.0.ProtoAlpha] — 2026-05-29

### GUI — темы и визуальный стиль

- Добавлены три режима темы: **Светлая / Тёмная / Авто** (следует системной теме Windows 11).
- Выбор темы определяется через реестр Windows (`AppsUseLightTheme`) без внешних зависимостей.
- Выбор сохраняется в `ui_config.json` и восстанавливается при следующем запуске.
- Выбор темы перенесён в меню **Вид → Тема**.
- При смене темы UI пересобирается с сохранением всего контекста (папки, кадры, настройки).
- Введены две полные палитры: `dark` (Adobe Photoshop–стиль) и `light` (Windows 11 светлая).
- Верхнее меню заменено на кастомную тёмную строку, чтобы Windows не рисовал белый native-menubar.
- Выпадающие меню переписаны на кастомные тёмные popup-окна: тёмная обводка, серый hover, тёмные разделители.
- Для popup-списков `Listbox` / `Combobox` добавлены тёмные цвета и убрана белая фокусная обводка.
- Тёмная DWM-шапка и favicon теперь применяются не только к главному окну, но и ко всем `Toplevel`-диалогам.

### GUI — ползунки (X / Y / Масштаб / Поворот)

- Ползунки полностью переписаны на custom Canvas вместо `ttk.Scale`.
- Тёмный трек, акцентная заливка активной части, светло-серый круглый thumb — всё чётко видно.
- Поддержка прокрутки колёсиком мыши (Shift × 10 для грубой настройки).
- Thumb и трек не сливаются с фоном ни в светлой, ни в тёмной теме.

### GUI — миниатюры кадров

- Починены галочки выбора кадров для экспорта: непомеченные теперь имеют видимый серый контур.
- Помеченный кадр — синий квадрат с белой галкой; снятый — прозрачный с чётким бордером.

### GUI — прочее

- Фавиконка приложения (`assets/image/favicon.ico`) устанавливается через `iconbitmap`.
- Шапка разделена на верхнюю строку меню и toolbar (путь + кнопки действий).
- Убрана кнопка «⬇ Дроп» (временно не работала).
- Панель цветокора: кнопки **Сохранить**, **Сбросить**, **Применить к папке** объединены в одну строку.
- Максимальный масштаб увеличен с 2.00 до 3.00.
- При загрузке новой папки первый подкаталог с исходниками открывается автоматически.
- Исправлено падение потоков `subprocess` на Windows из-за декодирования stdout/stderr в `cp1251`.
- Исправлена ошибка запуска GUI из-за дублирования параметра `relief` у `tk.Menu`.
- Окна **Zona**, **Дроп файлов**, **О программе**, **Инструкция**, **Что изменилось**, **Проверка обновлений**, **Горячие клавиши** получают иконку приложения вместо Python/Tk.

### Сборка (PyInstaller)

- Исправлена ошибка `Importing the numpy C-extensions failed` в собранном exe.
- Заменено `collect_submodules("numpy")` на `collect_all("numpy")` — теперь включаются `.pyd`/`.dll`.
- То же изменение применено для `PIL`.
- Иконка exe задаётся через поле `icon=` в `.spec`-файле.
- Папка `assets/` добавлена в список копируемых директорий при сборке.

### Файлы

- `assets/image/favicon.ico` — иконка окна и exe-файла.
- `assets/image/favicon.png` — PNG-версия иконки.
- `ui_config.json` — хранит последний выбор темы (создаётся при первом запуске).

## [0.0.0.2.ProtoAlpha] — 2026-05-26

### Кадрирование и редактор

- Единая система координат смещения X/Y для превью и экспорта (холст 1400×1050).
- Экспорт рендерит тот же рабочий кадр, что в редакторе.

- Поворот без чёрных углов: берётся дополнительная область исходника, без принудительного зума.
- Смещение X/Y корректно работает после поворота (экранные оси → оси исходника).

### GUI

- Drag-and-drop через `windnd` (без нестабильного WinAPI subclassing).
- Отдельное окно **«Дроп-окно»** для перетаскивания папки/файла.
- Поддержка дропа файла (берётся родительская папка).
