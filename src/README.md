# AutoRAW Compressor — техническое описание

**Версия: 0.0.1.13.ProtoAlpha** (источник: [`version.py`](version.py))

Прототип массового автокадрирования для съемки кроссовок.

## Что делает

- Читает файлы из папки `test` или другой указанной папки.
- Для RAW/NEF/DNG сначала вытаскивает самый большой встроенный JPEG из файла и уменьшает рабочее превью до `1200 px` по длинной стороне. Это дает нормальное качество предпросмотра/экспорта и не требует полной RAW-конвертации.
- Находит товар на светлом фоне.
- Применяет правила из папки `rules`:
  - `01` - `rules/1.jpg`: ширина товара 965 px, отступ снизу 185 px;
  - `02`, `03`, `04`, `08` - `rules/2-3-4-8.jpg`: ширина товара 965 px, отступ снизу 265 px;
  - `06` - `rules/6.jpg`: высота товара 897 px, отступы сверху/снизу 76 px, центр по X;
  - `05`, `07` - `manual_only`, как указано в `rules/other.png`.
- Считает crop в нормализованных координатах `0..1`, чтобы потом применить его к полному RAW-размеру в Photoshop.
- Сохраняет debug-картинки с рамками:
  - красная рамка: найденный товар;
  - синяя рамка: итоговый crop.
- Сохраняет `crop_plan.json` и заготовку `crop_plan.jsx`.

## Сборка dist (portable)

Из корня проекта:

```text
build.bat build
```

или `build.bat` → пункт **1**. Результат: `dist\AutoRAWCompressor\` — exe, `reference`, `rules`, `run_gui.bat`. Папку можно переносить на любой диск.

Зависимости сборки: `pip install -r requirements-build.txt`

Очистка: `build.bat clean`

Remote: `gitverse` → https://gitverse.ru/delbraun/AutoRAWCompressor

1. Создайте **пустой** репозиторий на [gitverse.ru/new](https://gitverse.ru/new): имя `AutoRAWCompressor`, без README и `.gitignore` (или используйте уже созданный: [delbraun/AutoRAWCompressor](https://gitverse.ru/delbraun/AutoRAWCompressor)).

```text
sync_gitverse.bat
```

Или с API-токеном (Настройки → Управление токенами → Репозитории + Публичное API):

## Запуск (разработка)

Из папки `Compressor`:

```text
setup.bat
run_autocrop.bat
```

или вручную:

```text
python src\autoraw_crop.py --input test --reference reference\Sneakers --output output
```

GUI:

```powershell
run_gui.bat
```

Можно перетащить корневую папку прямо на `run_gui.bat`, либо вставить путь в верхнюю строку и нажать `Загрузить папку`. Программа найдет внутри все папки с исходниками.

- слева появится дерево найденных папок;
- при поиске, загрузке превью и экспорте показывается прогресс: текущий файл, количество обработанных элементов и примерное оставшееся время;
- после загрузки корня программа показывает дерево папок, но не открывает первую папку автоматически, чтобы интерфейс не подвисал на тяжелых RAW;
- чекбокс напротив папки включает/исключает ее из экспорта;
- при выборе папки справа открываются 8 превью этой папки;
- чекбокс напротив превью включает/исключает конкретный кадр из экспорта;
- выберите кадр справа;
- двигайте товар мышью на большом превью;
- колесом мыши меняйте масштаб;
- ползунками можно точно поправить X/Y, масштаб и поворот;
- синие линии показывают правила для кадров `01`, `02`, `03`, `04`, `06`, `08`;
- `Экспорт отмеченных` создаст подпапку внутри каждой отмеченной папки с таким же именем, как у этой папки.

Если используете обычный Python, нужны только:

```text
Pillow
numpy
```

## Результаты

В папке `output` появятся:

- `*.preview.jpg` - легкие превью исходников;
- `*.debug.jpg` - превью с рамками распознавания;
- `*.layout.jpg` - быстрый предпросмотр финальной посадки на холсте `1400x1050`;
- `crop_plan.json` - план кадрирования;
- `crop_plan.jsx` - заготовка для Photoshop.

## Важно

Сейчас это первый рабочий прототип детектора. Он намеренно не конвертирует RAW сам, чтобы не убивать память. Следующий шаг - связать `crop_plan.json` с Photoshop-скриптом/экшеном, который будет открывать RAW и применять нормализованный crop к полному изображению.
